## 1) 시간 복잡도 구하는 방법(실전 기준)

### (1) 입력 크기 \(n\) 정의하기
- 배열 길이, 정점/간선 수(\(V, E\)), 문자열 길이 등 “무엇이 커질 때 느려지는지”를 먼저 정합니다.
- 예: 정렬은 보통 \(n=\) 원소 개수, 그래프는 \(V,E\)를 함께 씀.

### (2) 기본 연산을 세고 “가장 큰 항”만 남기기
- 대략적인 연산 횟수 \(T(n)\)을 세고,
- 상수/작은 항은 버리고 증가율이 가장 큰 항만 남깁니다.
  - 예: \(3n^2 + 10n + 50 \Rightarrow O(n^2)\)

### (3) 반복문 패턴
- **단일 반복문**
  ```text
  for i in 1..n => O(n)
  ```
- **중첩 반복문**
  ```text
  for i in 1..n
    for j in 1..n  => O(n^2)
  ```
- **반복 범위가 줄어드는 중첩**
  ```text
  for i in 1..n
    for j in 1..i  => 1+2+...+n = O(n^2)
  ```
- **반씩 줄어드는 루프**
  ```text
  while n > 1: n /= 2  => O(log n)
  ```

### (4) 조건문
- 보통 **최악의 경우(worst-case)**를 기준으로 잡는 경우가 많습니다.
- 다만 평균/기대 시간(average/expected) 또는 상환 분석(amortized)을 쓰기도 합니다.

### (5) 재귀는 점화식으로
- 예: 이분 탐색
  \[
  T(n) = T(n/2) + O(1) \Rightarrow O(\log n)
  \]
- 예: 병합 정렬
  \[
  T(n) = 2T(n/2) + O(n) \Rightarrow O(n\log n)
  \]
  (마스터 정리 등으로 풉니다)

### (6) 여러 단계가 이어지면 “더 큰 복잡도가 지배”
- 예: \(O(n\log n)\) 정렬 후 \(O(n)\) 스캔 → 전체 \(O(n\log n)\)

## 2) 최적화로 시간 복잡도를 줄인 대표 예시들

아래는 “많이 쓰이는 문제/알고리즘”에서 **발상을 바꿔 복잡도를 확 줄이는** 전형적인 사례입니다.

---

### 예시 A) 부분합(구간 합): 매번 더하기 → Prefix Sum
**문제:** 배열에서 구간 \([l,r]\) 합을 여러 번 질의

- **나이브:** 질의마다 직접 더하면  
  - 한 번 질의: \(O(n)\) (구간 길이만큼)
  - 질의가 \(q\)개면: \(O(qn)\)

- **최적화:** 누적합(prefix sum) \(P[i]=a[1]+...+a[i]\) 미리 계산
  - 전처리: \(O(n)\)
  - 질의: \(P[r]-P[l-1]\) → \(O(1)\)
  - 전체: \(O(n+q)\)

**핵심 아이디어:** “반복되는 계산을 미리 저장해 재사용(전처리/메모이제이션)”

---

### 예시 B) 소수 판별: 나눠보기 → \(\sqrt{n}\)까지만 / 에라토스테네스의 체
1) **단일 수 \(N\) 소수 판별**
- **나이브:** \(2..N-1\)로 나누기 → \(O(N)\)
- **개선:** \(2..\lfloor\sqrt{N}\rfloor\)까지만 검사 → \(O(\sqrt{N})\)

2) **1..N 범위의 모든 소수**
- **나이브:** 각 수를 판별하면 대략 \(O(N\sqrt{N})\) 수준
- **체(Sieve):** 배수 지우기 → \(O(N\log\log N)\) (전형적 분석 결과)

**핵심 아이디어:** “불필요한 검사를 수학적으로 줄이기 / 한 번에 여러 값을 처리”

---

### 예시 C) 최단 경로: 다익스트라 구현 최적화(자료구조 변경)
**문제:** 가중치(음수 없음) 그래프 최단경로

- **나이브(배열로 최소 거리 정점 찾기):**
  - 매 단계마다 최소 거리 정점을 선형 탐색 → \(O(V)\)  
  - 총 \(V\)번 반복 → \(O(V^2)\) (+ 간선 완화)
  - 밀집 그래프에선 괜찮지만, 희소 그래프에선 비효율

- **최적화(우선순위 큐/힙 사용):**
  - 최소 정점 추출 \(O(\log V)\)
  - 간선 완화 시 갱신 \(O(\log V)\)
  - 전체 \(O((V+E)\log V)\)

**핵심 아이디어:** “병목 연산(최솟값 찾기)을 더 좋은 자료구조로 바꾸기”

---

### 예시 D) 정렬: 버블/삽입(최악 \(O(n^2)\)) → 병합/힙/퀵(평균 \(O(n\log n)\))
- **나이브:** 버블 정렬/선택 정렬: \(O(n^2)\)
- **개선:** 병합 정렬/힙 정렬: \(O(n\log n)\) (퀵정렬은 평균 \(O(n\log n)\))

**핵심 아이디어:** “비교 정렬의 하한(대부분 \(n\log n\))에 가까운 알고리즘 선택”

---

### 예시 E) DP 최적화: 피보나치 재귀 → 메모이제이션/반복
- **나이브 재귀:**
  \[
  F(n)=F(n-1)+F(n-2)
  \]
  중복 호출 때문에 \(O(2^n)\)

- **최적화 1: 메모이제이션/DP**
  - 각 \(F(k)\)를 한 번만 계산 → \(O(n)\)

- **최적화 2: 공간 최적화**
  - 이전 두 값만 필요 → 시간 \(O(n)\), 공간 \(O(1)\)

**핵심 아이디어:** “중복 부분 문제 제거(DP) + 필요 상태만 유지”

## 3) 요약: 복잡도 줄이는 대표 전략
- **중복 계산 제거:** DP, 메모이제이션, prefix sum
- **자료구조 교체:** 힙/해시/균형트리로 병목 연산 개선
- **수학적 관찰:** \(\sqrt{n}\)까지만 검사, 체, 투 포인터 등
- **알고리즘 선택:** \(n^2 \to n\log n\) 같은 급의 개선

# 시간 복잡도 계산 방법과 최적화 사례

## 1. 시간 복잡도 구하는 방법

### 기본 원칙

```
1. 상수는 무시한다: O(2n) → O(n)
2. 최고차항만 남긴다: O(n² + n) → O(n²)
3. 최악의 경우를 기준으로 한다
```

### 단계별 분석 방법

#### Step 1: 기본 연산 세기

```python
# 예시 코드
def example(n):
    total = 0           # 1번
    for i in range(n):  # n번 반복
        total += i      # 1번 × n = n번
    return total        # 1번

# 총 연산: 1 + n + 1 = n + 2 → O(n)
```

#### Step 2: 반복문 분석 패턴

```python
# 패턴 1: 단일 반복문 → O(n)
for i in range(n):
    print(i)

# 패턴 2: 중첩 반복문 → O(n²)
for i in range(n):
    for j in range(n):
        print(i, j)

# 패턴 3: 중첩이지만 독립적 → O(n + m)
for i in range(n):
    print(i)
for j in range(m):
    print(j)

# 패턴 4: 절반씩 줄어듦 → O(log n)
i = n
while i > 0:
    print(i)
    i = i // 2

# 패턴 5: 반복 × 로그 → O(n log n)
for i in range(n):      # n번
    j = n
    while j > 0:        # log n번
        j = j // 2
```

#### Step 3: 재귀 함수 분석

```python
# 예시: 피보나치 (비효율적)
def fib(n):
    if n <= 1:
        return n
    return fib(n-1) + fib(n-2)

# 분석: T(n) = T(n-1) + T(n-2) + O(1)
# 호출 트리가 2배씩 증가 → O(2ⁿ)
```

### 시간 복잡도 비교표

```
입력 크기 n=1000일 때 대략적인 연산 횟수:

O(1)        →  1
O(log n)    →  10
O(n)        →  1,000
O(n log n)  →  10,000
O(n²)       →  1,000,000
O(n³)       →  1,000,000,000
O(2ⁿ)       →  우주가 끝날 때까지...
```

## 2. 최적화를 통해 시간 복잡도를 줄인 사례

### 사례 1: 정렬 알고리즘

```
버블 정렬 O(n²) → 병합 정렬 O(n log n)
```

```python
# ❌ 버블 정렬: O(n²)
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        for j in range(n - 1 - i):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
    return arr

# ✅ 병합 정렬: O(n log n)
def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])    # T(n/2)
    right = merge_sort(arr[mid:])   # T(n/2)
    
    return merge(left, right)       # O(n)

def merge(left, right):
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    result.extend(left[i:])
    result.extend(right[j:])
    return result

# 분석: T(n) = 2T(n/2) + O(n) → O(n log n)
```

**최적화 핵심**: 분할 정복 (Divide & Conquer)

---

### 사례 2: 탐색 알고리즘

```
선형 탐색 O(n) → 이진 탐색 O(log n)
```

```python
# ❌ 선형 탐색: O(n)
def linear_search(arr, target):
    for i, value in enumerate(arr):
        if value == target:
            return i
    return -1

# ✅ 이진 탐색: O(log n) - 정렬된 배열 필요
def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    
    while left <= right:
        mid = (left + right) // 2
        
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1    # 절반 제거
        else:
            right = mid - 1   # 절반 제거
    
    return -1

# 분석: 매 단계마다 탐색 범위가 절반 → O(log n)
```

**최적화 핵심**: 정렬된 데이터의 특성 활용

### 사례 3: 피보나치 수열

```
재귀 O(2ⁿ) → 메모이제이션 O(n) → 반복문 O(n) [공간 O(1)]
```

```python
# ❌ 단순 재귀: O(2ⁿ)
def fib_recursive(n):
    if n <= 1:
        return n
    return fib_recursive(n-1) + fib_recursive(n-2)

# ✅ 메모이제이션: O(n) 시간, O(n) 공간
def fib_memo(n, memo={}):
    if n in memo:
        return memo[n]
    if n <= 1:
        return n
    memo[n] = fib_memo(n-1, memo) + fib_memo(n-2, memo)
    return memo[n]

# ✅✅ 반복문: O(n) 시간, O(1) 공간
def fib_iterative(n):
    if n <= 1:
        return n
    prev, curr = 0, 1
    for _ in range(2, n + 1):
        prev, curr = curr, prev + curr
    return curr
```

```
호출 횟수 비교 (n=40):
- 재귀: 약 3억 번
- 메모이제이션: 40번
- 반복문: 39번
```

**최적화 핵심**: 중복 계산 제거 (Dynamic Programming)

### 사례 4: 중복 검사

```
이중 반복 O(n²) → 해시셋 O(n)
```

```python
# ❌ 브루트 포스: O(n²)
def has_duplicate_brute(arr):
    n = len(arr)
    for i in range(n):
        for j in range(i + 1, n):
            if arr[i] == arr[j]:
                return True
    return False

# ✅ 해시셋 활용: O(n)
def has_duplicate_hash(arr):
    seen = set()
    for num in arr:
        if num in seen:      # O(1) 평균
            return True
        seen.add(num)        # O(1) 평균
    return False
```

**최적화 핵심**: 해시 자료구조로 탐색 O(n) → O(1)

### 사례 5: 두 수의 합 찾기

```
브루트 포스 O(n²) → 해시맵 O(n)
```

```python
# ❌ 브루트 포스: O(n²)
def two_sum_brute(nums, target):
    n = len(nums)
    for i in range(n):
        for j in range(i + 1, n):
            if nums[i] + nums[j] == target:
                return [i, j]
    return []

# ✅ 해시맵: O(n)
def two_sum_hash(nums, target):
    num_to_index = {}
    for i, num in enumerate(nums):
        complement = target - num
        if complement in num_to_index:
            return [num_to_index[complement], i]
        num_to_index[num] = i
    return []
```

**최적화 핵심**: "찾아야 할 값"을 미리 저장

### 사례 6: 최단 경로 알고리즘

```
벨만-포드 O(VE) → 다익스트라 + 힙 O((V+E) log V)
```

```python
import heapq

# ✅ 다익스트라 (우선순위 큐): O((V+E) log V)
def dijkstra(graph, start):
    distances = {node: float('inf') for node in graph}
    distances[start] = 0
    pq = [(0, start)]  # (거리, 노드)
    
    while pq:
        current_dist, current = heapq.heappop(pq)  # O(log V)
        
        if current_dist > distances[current]:
            continue
        
        for neighbor, weight in graph[current].items():
            distance = current_dist + weight
            if distance < distances[neighbor]:
                distances[neighbor] = distance
                heapq.heappush(pq, (distance, neighbor))  # O(log V)
    
    return distances
```

**최적화 핵심**: 우선순위 큐로 최소값 추출 O(V) → O(log V)

## 3. 최적화 기법 요약

| 기법 | 원리 | 효과 |
|------|------|------|
| **분할 정복** | 문제를 절반씩 나눔 | O(n²) → O(n log n) |
| **해시 테이블** | 탐색을 O(1)로 | O(n) → O(1) |
| **메모이제이션** | 중복 계산 제거 | O(2ⁿ) → O(n) |
| **이진 탐색** | 정렬 데이터 활용 | O(n) → O(log n) |
| **우선순위 큐** | 효율적 최소/최대 추출 | O(n) → O(log n) |
| **투 포인터** | 양쪽에서 좁혀감 | O(n²) → O(n) |

### 🅰️ GPT-5.2
시간 복잡도와 **공간 복잡도**는 둘 다 “입력 크기 \(n\)이 커질 때 자원이 얼마나 늘어나는가”를 Big‑O로 나타내지만, 측정하는 자원이 다릅니다.

## 1) 공간 복잡도란?
- 알고리즘이 실행되는 동안 **추가로 필요한 메모리(공간)**가 입력 크기 \(n\)에 따라 얼마나 증가하는지.
- 보통 Big‑O로 \(O(1), O(n), O(n^2)\)처럼 표현합니다.

## 2) 시간 복잡도 vs 공간 복잡도 차이
- **시간 복잡도:** 연산/실행 시간 증가율 (CPU 사용량)
- **공간 복잡도:** 메모리 사용량 증가율 (RAM 사용량)

둘은 트레이드오프 관계인 경우가 많습니다.  
예: 미리 저장(메모이제이션)하면 **시간은 줄지만 공간은 늘어남**.

## 3) 공간 복잡도에서 “무엇을 세는지” (실전 기준)
공간은 보통 아래를 구분해 생각합니다.

### (1) 입력 공간(Input space)
- 입력 자체(배열, 문자열 등)를 저장하는 공간.
- 문제/상황에 따라 “입력은 이미 주어져 있으니 제외”하고 **추가 공간만** 따로 보는 경우가 많습니다.

### (2) 추가 공간(Auxiliary space)
- 알고리즘이 처리 중에 새로 쓰는 메모리:
  - 보조 배열, 해시맵, 큐/스택, 방문 배열
  - 재귀 호출 스택(콜 스택)

많은 교재/면접에서는 공간 복잡도를 말할 때 **추가 공간(보조 공간)**을 주로 의미합니다.

## 4) 예시로 감 잡기

### 예시 A: 배열 합 구하기
- 한 번 순회하며 합만 누적
- 시간: \(O(n)\)
- 추가 공간: 합 변수 하나 → \(O(1)\)

### 예시 B: Prefix Sum으로 구간합 빠르게
- 누적합 배열 \(P[1..n]\) 생성
- 시간(전처리): \(O(n)\), 질의: \(O(1)\)
- 추가 공간: 누적합 배열 → \(O(n)\)
- 즉, **시간 최적화를 위해 공간을 더 씀**

### 예시 C: 병합정렬 vs 퀵정렬(대표적인 트레이드오프)
- 병합정렬: 시간 \(O(n\log n)\), 추가 공간 보통 \(O(n)\) (병합용 배열)
- 퀵정렬: 평균 시간 \(O(n\log n)\), 추가 공간 보통 \(O(\log n)\) (재귀 스택; 최악 \(O(n)\))

### 예시 D: 재귀 DFS의 공간
- DFS를 재귀로 하면, 최악의 경우 호출 깊이만큼 스택 사용
- 그래프/트리 깊이가 \(n\)까지 갈 수 있으면 공간 \(O(n)\)

## 5) 정리
- 시간 복잡도는 “얼마나 오래 걸리나”
- 공간 복잡도는 “메모리를 얼마나 더 쓰나”
- 보통 공간 복잡도는 **추가(보조) 메모리** 기준으로 말하는 경우가 많고,
- 최적화는 종종 **시간 ↔ 공간**의 교환으로 이루어집니다.

원하면, 네가 떠올리는 특정 코드/알고리즘(예: BFS/DFS, 다익스트라, DP, 정렬)을 하나 주면 그 사례에서 시간·공간 복잡도를 각각 어떻게 계산하는지 같이 계산해줄게.

# 시간 복잡도 vs 공간 복잡도

## 한 줄 정리

```
시간 복잡도: 얼마나 "오래" 걸리는가? (연산 횟수)
공간 복잡도: 얼마나 "많은 메모리"를 쓰는가? (저장 공간)
```

## 직관적인 비유

```
요리로 비유하면:

시간 복잡도 = 요리하는 데 걸리는 시간
공간 복잡도 = 필요한 조리대 크기 (그릇, 도구 공간)

예: 100인분 요리
- 한 명씩 만들기 → 시간 오래, 공간 적음
- 한꺼번에 만들기 → 시간 짧음, 공간 많이 필요
```

## 코드로 비교

### 예시 1: 배열 뒤집기

```python
# 방법 1: 새 배열 사용
def reverse_new_array(arr):
    n = len(arr)
    result = [0] * n          # n개 공간 추가 할당
    for i in range(n):
        result[i] = arr[n - 1 - i]
    return result

# 시간: O(n) - n번 반복
# 공간: O(n) - 새 배열 n개


# 방법 2: 제자리(in-place) 교환
def reverse_in_place(arr):
    left, right = 0, len(arr) - 1
    while left < right:
        arr[left], arr[right] = arr[right], arr[left]
        left += 1
        right -= 1
    return arr

# 시간: O(n) - n/2번 반복 → O(n)
# 공간: O(1) - 변수 2개만 사용 (left, right)
```

**결과**: 같은 시간 복잡도지만, 방법 2가 공간 효율적

### 예시 2: 피보나치 수열

```python
# 방법 1: 메모이제이션 (재귀 + 저장)
def fib_memo(n, memo={}):
    if n in memo:
        return memo[n]
    if n <= 1:
        return n
    memo[n] = fib_memo(n-1, memo) + fib_memo(n-2, memo)
    return memo[n]

# 시간: O(n)
# 공간: O(n) - memo 딕셔너리 + 재귀 호출 스택


# 방법 2: 반복문
def fib_iterative(n):
    if n <= 1:
        return n
    prev, curr = 0, 1
    for _ in range(2, n + 1):
        prev, curr = curr, prev + curr
    return curr

# 시간: O(n)
# 공간: O(1) - 변수 2개만 사용
```

### 예시 3: 정렬 알고리즘 비교

```python
# 병합 정렬
def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])    # 새 배열 생성
    right = merge_sort(arr[mid:])   # 새 배열 생성
    return merge(left, right)       # 또 새 배열

# 시간: O(n log n)
# 공간: O(n) - 임시 배열 필요


# 퀵 정렬 (제자리)
def quick_sort(arr, low, high):
    if low < high:
        pivot = partition(arr, low, high)
        quick_sort(arr, low, pivot - 1)
        quick_sort(arr, pivot + 1, high)

# 시간: O(n log n) 평균
# 공간: O(log n) - 재귀 스택만 (제자리 정렬)
```

### 재귀의 공간 복잡도

```python
# 예시: 팩토리얼
def factorial(n):
    if n <= 1:
        return 1
    return n * factorial(n - 1)

# factorial(5) 호출 시 스택:
# factorial(5) → factorial(4) → factorial(3) → factorial(2) → factorial(1)
# 깊이 = n → 공간: O(n)
```

```
호출 스택 시각화:

factorial(5)
  └─ factorial(4)
       └─ factorial(3)
            └─ factorial(2)
                 └─ factorial(1) ← 여기서 반환 시작
                 
최대 깊이 = 5 = n → O(n) 공간
```

## 시간-공간 트레이드오프

### 핵심 개념

```
보통 시간을 줄이면 공간이 늘고,
공간을 줄이면 시간이 늘어난다.
```

### 실제 사례: 중복 문자 찾기

```python
# 공간 절약 → 시간 희생
def find_duplicate_space_efficient(s):
    for i in range(len(s)):
        for j in range(i + 1, len(s)):
            if s[i] == s[j]:
                return s[i]
    return None

# 시간: O(n²)
# 공간: O(1)


# 시간 절약 → 공간 희생
def find_duplicate_time_efficient(s):
    seen = set()
    for char in s:
        if char in seen:
            return char
        seen.add(char)
    return None

# 시간: O(n)
# 공간: O(n)
```

### 언제 뭘 선택?

```
메모리가 부족한 환경 (임베디드, 모바일)
→ 공간 복잡도 우선

실시간 응답이 중요한 환경 (웹 서버, 게임)
→ 시간 복잡도 우선

대부분의 경우
→ 시간 복잡도 우선 (메모리는 상대적으로 저렴)
```

## 주요 알고리즘 복잡도 비교

| 알고리즘 | 시간 | 공간 | 특징 |
|----------|------|------|------|
| 버블 정렬 | O(n²) | O(1) | 느리지만 추가 메모리 없음 |
| 병합 정렬 | O(n log n) | O(n) | 빠르지만 메모리 필요 |
| 퀵 정렬 | O(n log n) | O(log n) | 균형 잡힘 |
| 해시 탐색 | O(1) | O(n) | 빠르지만 저장 공간 필요 |
| 이진 탐색 | O(log n) | O(1) | 정렬 필요하지만 효율적 |
| DFS (재귀) | O(V+E) | O(V) | 스택 공간 사용 |
| BFS | O(V+E) | O(V) | 큐 공간 사용 |
