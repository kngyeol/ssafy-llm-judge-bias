> 코딩 문제
주어진 리스트에서 중복되지 않는 첫 번째 문제를 반환하는 함수를 작성해라.
함수명: find_unique_character
매개변수: characters (list)
반환값: 중복되지 않는 첫 번째 문자
예시:
입력: ['a', 'b', 'c', 'a', 'd']
출력: 'b'
입력: ['a', 'b', 'a', 'b', 'c']
출력: 'c'
입력: ['a', 'b', 'c', 'd', 'e']
출력: 'a'
언어는 자유롭게 사용 할수 있다.

### 🅰️ GPT-5.2
```python
def find_unique_character(characters):
    counts = {}
    for ch in characters:
        counts[ch] = counts.get(ch, 0) + 1

    for ch in characters:
        if counts[ch] == 1:
            return ch

    return None  # 중복되지 않는 문자가 없을 때


# 테스트
print(find_unique_character(['a', 'b', 'c', 'a', 'd']))  # 'b'
print(find_unique_character(['a', 'b', 'a', 'b', 'c']))  # 'c'
print(find_unique_character(['a', 'b', 'c', 'd', 'e']))  # 'a'
```

### 🅱️ Claude-Opus-4.5
# 중복되지 않는 첫 번째 문자 찾기

## 문제 분석
- 리스트에서 **한 번만 등장하는** 문자 중 **가장 먼저 나오는** 문자를 반환
- 등장 횟수가 1인 문자들 중 첫 번째 문자 찾기

---

## Python 풀이

### 방법 1: Counter 사용 (간결한 방법)

```python
from collections import Counter

def find_unique_character(characters):
    count = Counter(characters)
    
    for char in characters:
        if count[char] == 1:
            return char
    
    return None  # 중복되지 않는 문자가 없는 경우
```

### 방법 2: 딕셔너리 직접 구현 (라이브러리 없이)

```python
def find_unique_character(characters):
    # 각 문자의 등장 횟수 계산
    count = {}
    for char in characters:
        count[char] = count.get(char, 0) + 1
    
    # 첫 번째로 등장 횟수가 1인 문자 반환
    for char in characters:
        if count[char] == 1:
            return char
    
    return None
```

## 동작 과정 설명

**입력: `['a', 'b', 'c', 'a', 'd']`**

| 단계 | 설명 |
|------|------|
| 1️⃣ | 등장 횟수 계산: `{'a': 2, 'b': 1, 'c': 1, 'd': 1}` |
| 2️⃣ | 순서대로 확인: `'a'` → 2번 등장 ❌ |
| 3️⃣ | `'b'` → 1번 등장 ✅ → **반환** |

| 복잡도 | 값 | 설명 |
|--------|-----|------|
| 시간 | O(n) | 리스트를 2번 순회 |
| 공간 | O(n) | 카운트 딕셔너리 저장 |

WINNER: TIE
REASON: 두 답변 모두 첫 번째 비중복 문자를 O(n)으로 정확히 찾아 반환하고 유일 문자가 없을 때 None 처리까지 포함해 요구사항을 충족한다; 다만 B는 동일한 함수명을 두 번 정의해 한 구현이 다른 구현을 덮어써 혼동 소지가 있다.


## Turn 2

> 주어진 문제의 난이도를 높일 방법을 제시해라.

### 🅰️ GPT-5.2
- **입력 타입 확장**: `list`뿐 아니라 임의의 iterable(문자열, 제너레이터, 파일 스트림)에서도 동작하게 만들기. 단, 스트림은 “한 번만 읽을 수 있음” 제약이 생김.
- **반환 요구 변경**:  
  - 첫 번째 “문자”가 아니라 첫 번째 “유일 원소”(임의 객체) 반환  
  - 유일 원소가 없으면 예외를 던지거나, 인덱스를 반환하거나, `(원소, 인덱스)`를 반환하도록 요구
- **대소문자/정규화 처리**: 유니코드 정규화(NFC/NFD), 대소문자 무시(case-insensitive), 공백/특수문자 무시 같은 전처리 규칙 추가.
- **시간/공간 제약 부여**:  
  - 시간 O(n) 필수 + 공간 O(1) 요구(문자 집합 크기 제한 등 조건을 걸어 가능하게 만들기)  
  - 매우 큰 입력(수천만)에서 메모리 제한 하에 처리
- **단일 패스(One-pass) 요구**: “리스트 전체를 두 번 순회 금지” 또는 “카운팅 딕셔너리 사용 금지” 같은 제약을 걸어 대안 알고리즘을 고민하게 하기.
- **온라인(Streaming) 문제로 변경**: 입력이 실시간으로 들어올 때, 매 원소가 들어올 때마다 “현재까지의 첫 유일 문자”를 즉시 출력/질의할 수 있게 만들기(자료구조 설계: 큐+카운트+연결 리스트 등).
- **동점/우선순위 규칙 추가**: 유일 문자가 여러 개일 때 “가장 먼저 등장”이 아니라 “사전순 최소” 또는 “가중치가 가장 작은” 등 복합 기준 적용.
- **부분 구간 쿼리**: 여러 쿼리 `[l, r]`에 대해 해당 구간의 첫 유일 원소를 빠르게 답하기(전처리/세그먼트 트리/오프라인 알고리즘 등).
- **변경(update) 연산 포함**: 리스트 값이 바뀌는 업데이트가 있고, 매 업데이트 후 첫 유일 원소를 빠르게 반환(동적 자료구조).
- **병렬/분산 처리**: 입력을 여러 조각으로 나눠 처리한 뒤 합쳐서 전체의 첫 유일을 구하기(부분 결과를 어떻게 병합할지 설계).
- **에러/노이즈 처리**: 잘못된 타입, `None`, 빈 문자열, 멀티바이트 문자, 매우 긴 토큰 등 예외 케이스를 명세에 포함.
- **성능 측정 포함**: 특정 입력 크기에서 제한 시간/메모리 내 통과하도록 요구하고, 최적화(파이썬이라면 `collections.Counter` vs 수동 구현, 배열 인덱싱 등)까지 고려하게 만들기.

## 추천: Level 3 상세 풀이

가장 실용적이고 면접에서 자주 나오는 유형입니다:

```python
from collections import OrderedDict

class FirstUnique:
    def __init__(self):
        self.char_count = {}
        self.unique_order = OrderedDict()  # 순서 유지
    
    def add(self, char):
        if char in self.char_count:
            self.char_count[char] += 1
            if char in self.unique_order:
                del self.unique_order[char]  # 더 이상 유니크 아님
        else:
            self.char_count[char] = 1
            self.unique_order[char] = True
    
    def get_first_unique(self):
        if self.unique_order:
            return next(iter(self.unique_order))
        return None
```
