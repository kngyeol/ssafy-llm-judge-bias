# GPT vs Claude 50판, 그런데 심판이 셋이면 결과가 뒤집힌다

> SSAFY 크레딧 14,000점쯤 태워서 직접 돌려본 LLM 심판 편향 실험 기록

## TL;DR

같은 50개 챌린지, 같은 두 답변. 심판만 바꿨더니 GPT-5.2의 승률이 **72%(GPT-5 심판) → 42%(Claude 심판)** 로 30%p 차이가 났다. "LLM-as-a-judge"를 그냥 믿으면 안 되는 이유를 50판 데이터로 본다.

---

## 1. 왜 했나

요즘 평가 파이프라인 보면 죄다 LLM이 LLM을 채점한다. 챗봇 아레나, 자동 벤치마크, 심지어 RLHF 보상모델까지. 근데 진짜 궁금했다.

> "심판 모델이 자기랑 같은 family 답변을 더 좋아하는 거 아냐?"

논문은 많지만 직접 안 돌려봤으니 와닿지 않았다. SSAFY에서 받은 API 크레딧이 좀 남길래 한 판 붙여봤다.

## 2. 실험 설계

- **선수 A**: GPT-5.2 (`gpt-5.2`)
- **선수 B**: Claude-Opus-4.5 (`claude-opus-4-5-20251101`)
- **챌린지 50개**: code / writing / reasoning / creative / korean 각 10개
- **심판 3명**: GPT-5, Claude-Sonnet-4.5, Gemini-2.5-Flash
- **방식**: 같은 답변 쌍을 세 심판이 따로 채점 → A승 / B승 / TIE 중 하나
- **비용**: 약 14,000 크레딧 (SSAFY 지원 한도 내)

## 3. 충격적이었던 발견 — 같은 답인데 결론이 뒤집힌다

통계 표 보여주기 전에 실제 사례부터. 이게 이 글의 진짜 알맹이다.

### 케이스 1: `code-4` 괄호 검증 — "엄격함"이 미덕인가 "유연함"이 미덕인가

문제는 LeetCode식 괄호 매칭. 두 답변 거의 동일한 스택 구현이고 핵심 차이는 한 줄.

```python
# A (GPT-5.2): 괄호 아닌 문자 만나면 즉시 False
else:
    return False

# B (Claude): 괄호 아닌 문자는 그냥 무시
elif char in '([{':
    stack.append(char)
```

세 심판의 평결:

| 심판 | 승자 | 이유 (요약) |
|---|---|---|
| GPT-5 | A | "비괄호 문자 만나면 False로 처리해 더 견고" |
| Gemini | A | "타입 힌트 사용, 가독성과 유지보수성 높음" |
| **Claude** | **B** | "A는 괄호 아닌 문자에 즉시 False 반환 → 과도한 제약, B가 더 유연" |

같은 코드를 보고 GPT-5/Gemini는 "엄격한 게 좋다", Claude는 "유연한 게 좋다"고 정반대 결론을 낸다. 둘 다 일리는 있는데, 하필 family가 갈린다.

### 케이스 2: `code-5` SQL 두 번째 높은 급여 — Claude가 자기 family 버그를 알면서도 골랐다

이게 제일 흥미로웠다.

```sql
-- A (GPT-5.2)
SELECT (SELECT DISTINCT salary FROM Employee
        ORDER BY salary DESC LIMIT 1 OFFSET 1) AS SecondHighestSalary;

-- B (Claude)
SELECT MAX(salary) FROM Employee
WHERE salary < (SELECT MAX(salary) FROM Employee);
```

문제 요구는 "없으면 NULL". A는 서브쿼리로 감싸서 OK. B는 행이 1개뿐이면 NULL이 아니라 그 값 자체를 반환할 수 있다(엣지 버그).

| 심판 | 승자 | 이유 |
|---|---|---|
| GPT-5 | B | "표준 SQL이라 이식성 ↑, A는 LIMIT/OFFSET 방언" |
| Gemini | A | (A 손) |
| **Claude** | **A** | "**B는 급여가 1개뿐일 때 그 값을 반환하는 오류**, NULL이어야 함" |

Claude 심판은 자기 family인 B의 명백한 버그를 정확히 짚어냈다. 솔직했다. 그런데 같은 Claude 심판이 다른 라운드에선 자기 family 답이 100자 제한을 명백히 위반했는데도 그쪽 손을 들어준다 ↓

### 케이스 3: `writing-6` 환불 거절 — "100자 이내"라는 명확한 제약

문제: "한국어로 100자 이내". B(Claude) 답변:

> "고객님, 소중한 의견 감사드립니다. 안타깝게도 개봉 후 사용된 화장품은 위생상 교환·환불이 어려운 점 양해 부탁드립니다. 다음 구매 시 사용 가능한 10% 할인 쿠폰을 보내드릴게요. 추가 문의사항 있으시면 언제든 연락 주세요!"

세어보면 약 120자. 명백히 위반.

| 심판 | 승자 | 이유 |
|---|---|---|
| GPT-5 | A | "B는 100자 초과, 정책 언급 없음" |
| Gemini | B | (자세한 이유 없음) |
| **Claude** | **B** | "B는 거절 이유와 대안 제시. 다만 **글자 수 제한을 위반(약 120자)**" — 그런데도 B 선택 |

Claude 심판은 본인 입으로 "120자라 감점 불가피"라고 적어놓고 결론은 B. 채점자가 자기 모순에 빠진 순간이다.

### 케이스 4: `writing-1` SSAFY 한 줄 광고 — "설명문 vs 카피라이팅"

같은 문제, A는 "~프로그램이야" 같은 설명형, B는 "취뽀하자!" 같은 임팩트형.

| 심판 | 승자 | 한 줄 |
|---|---|---|
| GPT-5 | A | "5가지 톤 모두에서 SSAFY가 무엇인지 명확" |
| Gemini | A | "SSAFY가 '무엇인지' 일관성 있게 제시" |
| **Claude** | **B** | "광고 카피 특유의 임팩트, 행동 유도가 살아 있음" |

GPT 계열 심판은 "설명적"인 것을, Claude 심판은 "카피다움"을 더 쳐준다. 두 평가축이 다 정당한데 family에 따라 어느 축을 우선하는지가 갈린다.

### 케이스 5: `creative-2` 스타트업 이름 — 유머의 톤 평가

A(GPT)는 BugBell, RedPing 등 기능 중심 네이밍. B(Claude)는 GitGuilty, MyBadAlert 등 유머 중심.

| 심판 | 승자 |
|---|---|
| GPT-5 | A ("부정적 뉘앙스 감점") |
| **Claude** | **B** ("개발자 문화와 유머") |
| Gemini | B |

여기선 Gemini도 B 손을 들어줬다. 흥미롭게도 family flip 케이스 10개 중 압도적으로 많은 게 "Claude vs (GPT-5 + Gemini)" 구도였다.

### 케이스 6: `korean-6` 회식 문화 번역 — "교과서 vs 구어체"

같은 1차/2차/3차 설명. A는 "1차는 식사, 2차는 술집…" 안내문 톤. B는 "삼겹살집에서 시작해서… 밤새 노는 게 회식 문화야" 구어체.

| 심판 | 승자 | 이유 |
|---|---|---|
| GPT-5 | A | "B는 노래방·밤새 등 단정적 요소 임의 추가, 원문 충실도 ↓" |
| Gemini | A | "A가 더 명확한 설명 제공" |
| **Claude** | **B** | "B가 더 자연스러운 구어체, 문화적 맥락이 생생" |

같은 패턴이 5번 반복된다. **Claude는 "맛깔나는 표현"을, GPT는 "원문에 충실"을 우선한다.** 그리고 Claude-Opus가 대체로 더 맛깔나는 걸 쓴다(자기 family이기도 하고, Anthropic 트레이닝 스타일이기도 하고).

## 4. 데이터: 30%p 차이는 어디서 왔나

### 4.1 전체 스코어

| 심판 | A승 (GPT-5.2) | B승 (Claude) | TIE | A승률 |
|---|---|---|---|---|
| GPT-5 | **36** | 7 | 7 | **72%** |
| Claude-Sonnet | 21 | 13 | **16** | 42% |
| Gemini-Flash | 29 | 15 | 6 | 58% |

GPT 심판은 같은 family에 72%, Claude 심판은 42%. **30%p 차이는 그냥 우연이라 보기 어렵다.**

흥미로운 건 Claude 심판의 TIE 비율이 **32%(50판 중 16판)** 로 다른 둘의 거의 두 배라는 점. 특히 reasoning 카테고리는 **10판 중 9판이 TIE**다. 자기 family를 노골적으로 띄우진 않는데, "양쪽 다 비슷하다"고 빠지는 회피 편향이 강하다.

### 4.2 카테고리별 — 어디서 갈렸나

| 카테고리 | GPT-5 (A승) | Claude (A승) | Gemini (A승) | 격차 |
|---|---|---|---|---|
| writing | 10/10 | 8/10 | 5/10 | **5판** |
| reasoning | 6/10 | 1/10 | 4/10 | **5판** |
| code | 6/10 | 3/10 | 7/10 | 4판 |
| creative | 9/10 | 6/10 | 6/10 | 3판 |
| korean | 5/10 | 3/10 | 7/10 | 4판 |

- **writing**: GPT-5는 GPT-5.2 답에 100% 손 들어줌(10/10). 이 정도면 한쪽으로 쏠린 거다.
- **reasoning**: 객관적 답이 있는 영역인데도 GPT-5는 6승 vs Claude는 1승. "정답 4536개" 같은 케이스에서 GPT-5는 "1줄 형식 위반이라 A 승", Claude는 "본질 동일하니 TIE" — 형식주의 vs 본질주의가 갈렸다.
- **korean**: 한국어 영역만큼은 Claude 심판이 자기 family(B)에 6번 손 들어줬다. 한국어 표현력에선 실제로 Opus가 좀 낫긴 한데, 같은 답을 보고 Gemini는 7번이나 A 손을 들어준 걸 보면 이것도 결국 평가축의 차이다.

## 5. 한계 — 솔직하게

이 데이터로 "LLM 심판은 편향이 있다" 정도까진 말할 수 있는데, 그 이상은 조심해야 한다.

1. **표본 50개는 적다.** 카테고리당 10개면 통계적으로 흔들린다. 30%p 차이가 신뢰구간 안에 들어올 가능성을 부정할 수 없음. 제대로 하려면 챌린지 수를 5배는 늘려야 한다.
2. **심판 모델 자체가 동급이 아니다.** GPT-5(상위) vs Claude-Sonnet-4.5(중급) vs Gemini-2.5-**Flash**(소형). 만약 Claude-Opus를 심판으로 썼다면 결과가 또 달랐을 것. "심판 모델 격차"와 "family 편향"이 섞여 있다.
3. **챌린지 편향.** 50개 챌린지를 누가 만들었느냐(나)에 따라 GPT 스타일에 유리한 문제가 끼어들었을 수 있다. 특히 "1줄로", "100자 이내" 같은 형식 제약이 많아서 형식 엄격한 GPT-5 심판이 더 후하게 점수 줬을 가능성.
4. **단일 시드.** 같은 챌린지를 여러 번 돌려서 분산을 봐야 진짜인데, 크레딧 한계로 1회만 돌렸다.
5. **A/B 위치 편향**(position bias)은 따로 컨트롤하지 않았다. 심판 프롬프트에서 항상 GPT-5.2가 A로 들어갔으니 이것도 변수다.

그래도 "**같은 답인데 심판 family에 따라 결론이 뒤집힌 사례 10개 이상이 실제로 존재한다**"는 사실은 흔들리지 않는다. 통계가 약해도 케이스 스터디는 살아 있다.

## 6. 결론 — LLM 심판 신뢰하기 전에

세 줄로 줄이면:

1. **단일 LLM 심판으로 평가하지 마라.** 최소 3종 이상 family를 섞고, 다수결 또는 평균을 봐라.
2. **TIE 비율을 봐라.** Claude 심판처럼 reasoning에서 90% TIE 주는 심판은 신호를 죽이고 있을 가능성이 높다.
3. **심판 코멘트를 직접 읽어라.** 위에서 본 케이스 3처럼, 코멘트엔 "100자 위반"이라고 적어놓고 결론은 정반대인 경우가 실제로 있다. 숫자만 집계하면 놓친다.

LLM 심판은 분명히 유용한 도구다. 다만 인간 평가자처럼 신뢰성·재현성·편향을 항상 의심해야 한다. 이번 실험에서 가장 큰 발견은 "in-family bias"보다도 **"심판 모델마다 우선시하는 평가축이 다르다"** 였다. 형식 vs 본질, 엄격함 vs 유연함, 설명 vs 임팩트.
