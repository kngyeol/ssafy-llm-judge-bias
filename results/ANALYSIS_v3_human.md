# GPT vs Claude 50판 — v3: 사람이 직접 채점하니 LLM 심판이 다 틀렸다

> v1·v2는 LLM 심판들끼리 비교만 했다. v3에서 본인이 라벨 가린 채 50판 직접 채점해서 **ground truth**를 만들었다. 결과는 충격적이다.

## 1. 인간 vs LLM — A 선호율 비교

| 평가자 | A승 (GPT-5.2) | B승 (Claude) | TIE | A 선호율 (TIE 제외) |
|---|---|---|---|---|
| **🧑 인간 (본인)** | **12** | **19** | **19** | **39%** |
| GPT-5 심판 | 36 | 7 | 7 | 72% |
| Claude-Sonnet 심판 | 21 | 13 | 16 | 42% |
| Gemini-Flash 심판 | 29 | 15 | 6 | 58% |

핵심: **본인 눈엔 Claude-Opus가 GPT-5.2보다 좋았다** (19승 vs 12승). v1·v2 내내 "GPT가 이긴 줄 알았던 것"은 LLM 심판들의 환상이었다.

### 1.1 인간 대비 편향 정량화

| 심판 | A 선호율 | 인간 대비 |
|---|---|---|
| 인간 (기준) | 39% | — |
| **Claude-Sonnet** | **42%** | **+3%p ← 가장 정확** |
| Gemini-Flash | 58% | +19%p |
| **GPT-5** | **72%** | **+33%p ← 거대한 family bias** |

v2 position swap에서 "GPT-5의 72%는 진짜 family bias"라고 했는데, v3가 이를 직접 검증한다: GPT-5는 인간보다 정확히 **33%p만큼 자기 family를 더 좋아한다.**

## 2. 라운드별 일치율 — 모두가 인간과 어긋난다

같은 라운드에서 인간과 LLM 심판의 판정이 일치한 비율:

| 비교 | 일치 라운드 | 비율 |
|---|---|---|
| **심판들끼리** (LLM ↔ LLM) | | |
| GPT-5 ↔ Claude | 29/50 | 58% |
| GPT-5 ↔ Gemini | 31/50 | 62% |
| Claude ↔ Gemini | 22/50 | 44% |
| **심판 ↔ 인간** | | |
| GPT-5 ↔ Human | **15/50** | **30%** |
| Claude ↔ Human | 19/50 | 38% |
| Gemini ↔ Human | 19/50 | 38% |

**결정적 발견**: LLM 심판들은 *서로* 더 잘 맞고 (44-62%), 인간과는 일치율이 30-38%다. 3-class(A/B/TIE) 무작위 baseline이 33%니까 **GPT-5 심판은 사실상 무작위 수준**이다.

> LLM 심판은 "사람을 닮은 평가자"가 아니라 "서로를 닮은 편향된 평가자들의 군집"이다.

### 2.1 카테고리별 인간 일치율

| 카테고리 | GPT-5 | Claude-Sonnet | Gemini-Flash | 인간 분포 |
|---|---|---|---|---|
| code | 40% | 30% | 20% | A:2 B:3 TIE:5 |
| creative | 30% | 40% | **50%** | A:2 B:6 TIE:2 |
| korean | 20% | 30% | **40%** | A:5 B:2 TIE:3 |
| reasoning | 30% | **70%** | 20% | A:0 B:2 **TIE:8** |
| writing | 30% | 20% | **60%** | A:3 B:6 TIE:1 |

- **Reasoning에서 Claude 70% 일치**는 *통찰*이 아니라 *공통 회피*다. 인간이 TIE 8개, Claude도 TIE를 가장 많이 주는 심판. 둘 다 결정을 회피해서 일치한 것.
- **Writing/Creative는 Gemini가 가장 인간스러움** — 인간이 B(Claude)를 많이 좋아한 카테고리에서 Gemini도 그 방향을 따른다.
- **Code는 GPT-5가 1위지만 40%에 불과** — 코드 평가도 객관적이지 않다.

## 3. 만장일치 오류 — 모든 LLM이 한 방향으로 틀린 라운드들

LLM 심판 3개가 *모두* 같은 답을 골랐는데 인간이 다른 답을 고른 라운드:

### 3.1 LLM 3개 모두 A, 인간은 B/TIE (조직적 GPT 과대평가)

8개 라운드:

| ID | 인간 | 카테고리 | 비고 |
|---|---|---|---|
| `code-2` | B | code | 디버깅 — Claude의 풀이가 인간이 보기엔 더 적합 |
| `code-8` | TIE | code | 피보나치 메모이제이션 — 둘 다 같다고 봄 |
| `creative-3` | B | creative | Claude의 창의가 더 좋았음 |
| `creative-7` | TIE | creative | 동일 수준 |
| `creative-9` | B | creative | 동상 |
| `korean-5` | TIE | korean | 동일 수준 |
| `writing-3` | B | writing | Claude 글이 더 인상적 |
| `writing-7` | B | writing | 동상 |

### 3.2 LLM 3개 모두 B, 인간은 A

**단 1개**:

| ID | 인간 | 비고 |
|---|---|---|
| `korean-2` | A | 사자성어 추천 — Claude의 답에 결함이 있었던 듯 (v2 self-judge에서 GPT-5.2도 자기 답의 사자성어 오류를 인정한 라운드) |

**비대칭 8:1** — LLM 심판들의 만장일치 오류는 거의 모두 한 방향, GPT 과대평가다. 무작위 오류가 아니라 *체계적 편향*이라는 결정적 증거.

## 4. v1·v2·v3 종합 결론

| 버전 | 가설 | 데이터 | 결론 |
|---|---|---|---|
| v1 | "심판 family가 다르면 결과 다를까?" | 3 심판 50판 | "30%p 격차 발생, family bias 의심" |
| v2 | "그 격차가 진짜 family bias인가?" | + 통제 3종 | "GPT는 진짜 family bias, Gemini는 절반이 position bias, Claude는 회피 패턴" |
| **v3** | **"그래서 인간 기준 누가 맞나?"** | **+ 인간 직접 채점** | **"전부 틀렸다. Claude-Sonnet이 가장 가깝지만 그것도 30%대"** |

## 5. v3의 진짜 메시지

세 줄로 줄이면:

1. **LLM 심판들끼리의 합의는 합의의 정확성을 보장하지 않는다.** 그들은 *서로* 더 닮았을 뿐 인간과는 닮지 않았다. 3개 심판이 만장일치라도 인간 기준 틀린 경우가 9건/50 있었고, 그중 8건이 한 방향(GPT 과대평가).
2. **"가장 인간스러운 심판"은 family bias가 가장 적은 심판이다.** Claude-Sonnet이 인간과 가장 가까운 A 선호율(42% vs 인간 39%)을 보였다. v1에서 가장 회피적이라고 비판받던 심판이 사실 가장 정직했다.
3. **LLM-as-a-judge의 30~40% 정확도는 production에 못 쓰는 수준이다.** 평가 자동화를 LLM에 맡기려는 모든 파이프라인은 인간 baseline부터 측정해야 한다.

## 6. 한계

- **표본 1명(나)** — 다른 사람이 채점하면 다를 수 있음. 특히 한국어/창작은 취향 영향이 큼.
- **인간이 더 옳다는 보장은 없다** — 단지 다른 가설이다. 객관적 정답이 있는 코딩에서 인간이 LLM과 어긋난 게 7개 있었는데, 누가 맞는지는 케이스마다 봐야 한다.
- **인간도 피로 효과** — 50개 연속 채점에서 후반은 덜 신중했을 수 있음.
- **TIE를 자유롭게 쓸 수 있게 한 게 인간을 회피적으로 만들었을 수도.** 강제 binary 선택을 했다면 결과가 달랐을 가능성.

## 7. 다음 단계 — v4가 만든다면

- 다중 인간 채점자 (3~5명) → inter-rater reliability 측정
- 같은 50판에 대해 forced-binary (TIE 금지) 인간 채점
- "왜 그렇게 골랐는지" 인간이 코멘트도 작성 → LLM 심판 코멘트와 의미 비교
- 인간 baseline을 ground truth로 두고 LLM 심판들을 *재훈련*하는 prompt tuning 실험
