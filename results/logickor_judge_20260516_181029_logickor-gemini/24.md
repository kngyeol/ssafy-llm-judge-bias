## 1) 멀티프로세스 구현(기본 방법): `multiprocessing` 모듈

### 핵심 아이디어
- **프로세스(Process)** 는 OS 레벨에서 **각자 독립된 메모리 공간**을 가지는 실행 단위입니다.
- Python에서는 표준 라이브러리 **`multiprocessing`** 로 손쉽게 여러 프로세스를 띄울 수 있습니다.

### 대표적 구현 방법(예시)
```python
from multiprocessing import Process

def worker(n):
    print("work:", n)

if __name__ == "__main__":
    ps = [Process(target=worker, args=(i,)) for i in range(4)]
    for p in ps: p.start()
    for p in ps: p.join()
```

또는 풀(Pool)을 사용하는 방식도 흔합니다.
```python
from multiprocessing import Pool

def f(x):
    return x * x

if __name__ == "__main__":
    with Pool(4) as p:
        print(p.map(f, [1, 2, 3, 4]))
```

## 2) 멀티스레드 구현(기본 방법): `threading` 모듈

### 핵심 아이디어
- **스레드(Thread)** 는 같은 프로세스 안에서 실행되는 흐름으로, **메모리를 공유**합니다.
- Python 표준 라이브러리 **`threading`** 으로 구현합니다.

### 대표적 구현 방법(예시)
```python
import threading

def worker(n):
    print("work:", n)

threads = [threading.Thread(target=worker, args=(i,)) for i in range(4)]
for t in threads: t.start()
for t in threads: t.join()
```

# 두 방법의 차이점(자세히)

아래는 “왜/언제 프로세스를 쓰고, 왜/언제 스레드를 쓰는가”에 직결되는 핵심 차이들입니다.

## A. 메모리 구조(격리 vs 공유)
- **멀티프로세스**
  - 각 프로세스는 **메모리 공간이 분리**되어 서로의 변수/객체에 직접 접근 불가
  - 데이터 공유는 `Queue`, `Pipe`, `Manager`, `shared_memory` 등을 통한 **IPC(프로세스 간 통신)** 가 필요
  - 장점: 한 프로세스가 죽어도 다른 프로세스에 영향이 적고, 메모리 오염 위험이 낮음
- **멀티스레드**
  - 같은 프로세스 내에서 **메모리(힙)를 공유**
  - 공유 데이터에 동시 접근 시 **경쟁 상태(race condition)** 발생 가능 → `Lock`, `RLock`, `Semaphore`, `Condition` 등 동기화 필요
  - 장점: 메모리 공유가 자연스럽고 데이터 전달 비용이 적음

## B. 성능과 GIL 영향(특히 CPython)
- **멀티스레드(중요)**  
  - CPython에는 **GIL(Global Interpreter Lock)** 이 있어, 기본적으로 **한 프로세스에서 한 순간에 하나의 스레드만 Python 바이트코드를 실행**합니다.
  - 결과:
    - **CPU-bound(순수 계산 중심)** 작업에서는 스레드를 늘려도 병렬 계산 성능 향상이 제한적(오히려 문맥 전환 비용으로 느려질 수 있음)
    - **I/O-bound(네트워크, 디스크, 대기)** 작업에서는 스레드가 기다리는 동안 다른 스레드가 실행되어 **효율이 좋아짐**
- **멀티프로세스**
  - 프로세스는 GIL을 **각자 따로** 가지므로, 여러 코어에서 **진짜 병렬로 CPU-bound 작업을 수행** 가능
  - CPU 사용량을 코어 수만큼 끌어올리기 유리

정리:
- **CPU-bound ⇒ 보통 multiprocessing 유리**
- **I/O-bound ⇒ threading(또는 asyncio) 유리**

## C. 생성/전환 비용과 오버헤드
- **프로세스**
  - 생성 비용이 스레드보다 큼(프로세스 시작, 메모리 공간 분리 등)
  - IPC 비용(직렬화/역직렬화, 파이프/큐 전송 등)이 발생할 수 있음
- **스레드**
  - 생성/전환 비용이 비교적 작고 가벼움
  - 대신 동기화 비용(락 경합)과 버그 위험(데드락, 레이스)이 존재

## D. 안정성과 격리(장애 전파)
- **프로세스**
  - 한 프로세스가 크래시/메모리 누수/치명적 오류를 내도 다른 프로세스에 영향이 상대적으로 적음
  - 서버/배치에서 안정성을 위해 선호되기도 함
- **스레드**
  - 같은 프로세스 공간을 공유하므로, 심각한 오류(예: C 확장 모듈 문제)가 있으면 전체 프로세스가 영향을 받을 수 있음

## E. 데이터 공유의 편의성
- **스레드**
  - 전역 변수, 객체 등을 그대로 공유 가능(편리)
  - 그러나 동기화가 필요해 복잡해질 수 있음
- **프로세스**
  - 공유가 불편하지만(IPC 필요), 그만큼 설계가 명시적이고 격리가 됨

## F. 플랫폼 관련 주의(특히 Windows)
- `multiprocessing`는 OS에 따라 프로세스 시작 방식이 달라서(Windows는 기본이 `spawn`)  
  **`if __name__ == "__main__":` 가 거의 필수**입니다.
- 스레드는 이런 제약이 상대적으로 적습니다.

## 1. 멀티스레딩 (Threading)

### 개념
하나의 프로세스 내에서 여러 스레드가 **메모리를 공유**하며 동시에 실행됩니다.

```python
import threading
import time

# 공유 자원
counter = 0
lock = threading.Lock()

def worker(name, count):
    """스레드 작업 함수"""
    global counter
    for i in range(count):
        # Lock을 사용한 동기화 (Race Condition 방지)
        with lock:
            counter += 1
            print(f"[{name}] 작업 {i+1} 완료, counter = {counter}")
        time.sleep(0.1)  # I/O 작업 시뮬레이션

def main_threading():
    print("=== 멀티스레딩 예제 ===\n")
    
    # 스레드 생성
    threads = [
        threading.Thread(target=worker, args=(f"Thread-{i}", 3))
        for i in range(3)
    ]
    
    start_time = time.time()
    
    # 스레드 시작
    for t in threads:
        t.start()
    
    # 모든 스레드 완료 대기
    for t in threads:
        t.join()
    
    elapsed = time.time() - start_time
    print(f"\n완료! 최종 counter = {counter}")
    print(f"소요 시간: {elapsed:.2f}초")

if __name__ == "__main__":
    main_threading()
```

## 2. 멀티프로세싱 (Multiprocessing)

### 개념
여러 개의 **독립적인 프로세스**가 각자의 메모리 공간에서 실행됩니다.

```python
import multiprocessing as mp
import time
import os

def cpu_intensive_task(name, numbers):
    """CPU 집약적 작업 함수"""
    pid = os.getpid()
    print(f"[{name}] 시작 (PID: {pid})")
    
    # 실제 CPU 작업 (소수 계산)
    result = sum(is_prime(n) for n in numbers)
    
    print(f"[{name}] 완료 - {result}개의 소수 발견")
    return result

def is_prime(n):
    """소수 판별 함수"""
    if n < 2:
        return False
    for i in range(2, int(n**0.5) + 1):
        if n % i == 0:
            return False
    return True

def main_multiprocessing():
    print("=== 멀티프로세싱 예제 ===\n")
    print(f"메인 프로세스 PID: {os.getpid()}")
    print(f"CPU 코어 수: {mp.cpu_count()}\n")
    
    # 작업 데이터 분할
    all_numbers = list(range(1, 100001))
    chunk_size = len(all_numbers) // 4
    chunks = [
        all_numbers[i:i + chunk_size] 
        for i in range(0, len(all_numbers), chunk_size)
    ]
    
    start_time = time.time()
    
    # 프로세스 풀 사용
    with mp.Pool(processes=4) as pool:
        # 각 청크에 대해 병렬 처리
        results = pool.starmap(
            cpu_intensive_task,
            [(f"Process-{i}", chunk) for i, chunk in enumerate(chunks)]
        )
    
    elapsed = time.time() - start_time
    print(f"\n총 소수 개수: {sum(results)}")
    print(f"소요 시간: {elapsed:.2f}초")

if __name__ == "__main__":
    main_multiprocessing()
```

## 4. 성능 비교 실험

```python
import threading
import multiprocessing as mp
import time

def cpu_task(n):
    """CPU 집약적 작업: 피보나치 계산"""
    if n <= 1:
        return n
    a, b = 0, 1
    for _ in range(n - 1):
        a, b = b, a + b
    return b

def io_task(seconds):
    """I/O 집약적 작업: 대기"""
    time.sleep(seconds)
    return seconds

def benchmark(task_type="cpu"):
    """벤치마크 실행"""
    if task_type == "cpu":
        task = lambda: cpu_task(100000)
        task_name = "CPU 집약적"
    else:
        task = lambda: io_task(0.5)
        task_name = "I/O 집약적"
    
    num_tasks = 4
    
    # 1. 순차 실행
    start = time.time()
    for _ in range(num_tasks):
        task()
    sequential_time = time.time() - start
    
    # 2. 멀티스레딩
    start = time.time()
    threads = [threading.Thread(target=task) for _ in range(num_tasks)]
    for t in threads:
        t.start()
    for t in threads:
        t.join()
    threading_time = time.time() - start
    
    # 3. 멀티프로세싱
    start = time.time()
    with mp.Pool(num_tasks) as pool:
        pool.map(lambda x: task(), range(num_tasks))
    multiprocessing_time = time.time() - start
    
    print(f"\n{'='*50}")
    print(f"📊 {task_name} 작업 벤치마크 결과")
    print(f"{'='*50}")
    print(f"순차 실행:      {sequential_time:.3f}초")
    print(f"멀티스레딩:     {threading_time:.3f}초 "
          f"(속도 향상: {sequential_time/threading_time:.2f}x)")
    print(f"멀티프로세싱:   {multiprocessing_time:.3f}초 "
          f"(속도 향상: {sequential_time/multiprocessing_time:.2f}x)")

if __name__ == "__main__":
    benchmark("cpu")
    benchmark("io")
```

> multiprocessing을 활용하여 변수 a에 1을 1억 번 더하는 코드를 작성해보시오.

### 🅰️ GPT-5.2
```python
import multiprocessing as mp

def worker(n: int) -> int:
    a = 0
    for _ in range(n):
        a += 1
    return a

if __name__ == "__main__":
    TOTAL = 100_000_000
    nproc = mp.cpu_count()

    base = TOTAL // nproc
    rem = TOTAL % nproc
    counts = [base + (1 if i < rem else 0) for i in range(nproc)]

    with mp.Pool(processes=nproc) as pool:
        parts = pool.map(worker, counts)

    a = sum(parts)
    print("a =", a)  # 기대값: 100000000
```

설명(핵심만):
- 프로세스마다 로컬 변수 `a`를 증가시켜 결과를 반환하고,
- 메인 프로세스에서 각 프로세스의 결과를 `sum()`으로 합쳐 최종 `a = 100_000_000`을 만듭니다.

# Multiprocessing을 활용한 1억 번 덧셈

## 🎯 문제 분석

멀티프로세싱에서 변수를 공유하려면 **특별한 공유 메모리 객체**가 필요합니다.
각 프로세스는 독립된 메모리 공간을 가지므로 일반 변수는 공유되지 않습니다.

## 방법 1: `multiprocessing.Value` 사용 (공유 메모리)

```python
import multiprocessing as mp
import time

def add_with_lock(shared_value, lock, count):
    """Lock을 사용한 안전한 덧셈"""
    for _ in range(count):
        with lock:
            shared_value.value += 1

def add_without_lock(shared_value, count):
    """Lock 없이 덧셈 (Race Condition 발생!)"""
    for _ in range(count):
        shared_value.value += 1

def method1_shared_value():
    """방법 1: Value + Lock 사용"""
    print("=" * 50)
    print("방법 1: multiprocessing.Value + Lock")
    print("=" * 50)
    
    # 공유 변수 생성 ('i' = 정수형)
    a = mp.Value('i', 0)
    lock = mp.Lock()
    
    num_processes = 4
    count_per_process = 100_000_000 // num_processes  # 1억 ÷ 4
    
    processes = [
        mp.Process(target=add_with_lock, args=(a, lock, count_per_process))
        for _ in range(num_processes)
    ]
    
    start = time.time()
    
    for p in processes:
        p.start()
    for p in processes:
        p.join()
    
    elapsed = time.time() - start
    
    print(f"최종 결과: a = {a.value:,}")
    print(f"소요 시간: {elapsed:.2f}초")
    print(f"예상 결과: {100_000_000:,}")
    print()

if __name__ == "__main__":
    method1_shared_value()
```

⚠️ **문제점**: Lock으로 인해 실제로는 순차 실행과 비슷하게 느림

## 방법 2: 분할 계산 후 합산 (권장 ⭐)

```python
import multiprocessing as mp
import time

def partial_sum(count):
    """각 프로세스에서 부분 합계 계산"""
    local_sum = 0
    for _ in range(count):
        local_sum += 1
    return local_sum

def method2_divide_and_conquer():
    """방법 2: 분할 계산 후 합산 (가장 효율적)"""
    print("=" * 50)
    print("방법 2: 분할 계산 후 합산 (권장)")
    print("=" * 50)
    
    num_processes = 4
    count_per_process = 100_000_000 // num_processes
    
    start = time.time()
    
    # Pool을 사용하여 병렬 처리
    with mp.Pool(processes=num_processes) as pool:
        results = pool.map(partial_sum, [count_per_process] * num_processes)
    
    # 결과 합산
    a = sum(results)
    
    elapsed = time.time() - start
    
    print(f"각 프로세스 결과: {results}")
    print(f"최종 결과: a = {a:,}")
    print(f"소요 시간: {elapsed:.2f}초")
    print()

if __name__ == "__main__":
    method2_divide_and_conquer()
```

## 방법 3: `Array` 사용

```python
import multiprocessing as mp
import time

def add_to_array(shared_array, index, count):
    """배열의 특정 인덱스에 값 누적"""
    for _ in range(count):
        shared_array[index] += 1

def method3_shared_array():
    """방법 3: 공유 배열 사용"""
    print("=" * 50)
    print("방법 3: multiprocessing.Array")
    print("=" * 50)
    
    num_processes = 4
    count_per_process = 100_000_000 // num_processes
    
    # 공유 배열 생성 (각 프로세스가 자신의 인덱스에 저장)
    shared_array = mp.Array('i', num_processes)
    
    processes = [
        mp.Process(target=add_to_array, args=(shared_array, i, count_per_process))
        for i in range(num_processes)
    ]
    
    start = time.time()
    
    for p in processes:
        p.start()
    for p in processes:
        p.join()
    
    elapsed = time.time() - start
    
    a = sum(shared_array)
    
    print(f"배열 값: {list(shared_array)}")
    print(f"최종 결과: a = {a:,}")
    print(f"소요 시간: {elapsed:.2f}초")
    print()

if __name__ == "__main__":
    method3_shared_array()
```

## 📊 전체 비교 코드

```python
import multiprocessing as mp
import time

# ============ 작업 함수들 ============

def partial_sum(count):
    """부분 합계 계산"""
    result = 0
    for _ in range(count):
        result += 1
    return result

def add_with_lock(shared_val, lock, count):
    """Lock 사용 덧셈"""
    for _ in range(count):
        with lock:
            shared_val.value += 1

def add_to_array(arr, idx, count):
    """배열 인덱스에 누적"""
    for _ in range(count):
        arr[idx] += 1

# ============ 벤치마크 ============

def benchmark_all():
    TOTAL = 100_000_000  # 1억
    NUM_PROC = mp.cpu_count()
    CHUNK = TOTAL // NUM_PROC
    
    print(f"총 덧셈 횟수: {TOTAL:,}")
    print(f"프로세스 수: {NUM_PROC}")
    print(f"프로세스당 횟수: {CHUNK:,}\n")
    
    results = {}
    
    # 1. 순차 실행 (기준)
    print("[1/4] 순차 실행...")
    start = time.time()
    a = 0
    for _ in range(TOTAL):
        a += 1
    results['순차 실행'] = time.time() - start
    print(f"     결과: {a:,}, 시간: {results['순차 실행']:.2f}초\n")
    
    # 2. Pool + map (권장)
    print("[2/4] Pool + map...")
    start = time.time()
    with mp.Pool(NUM_PROC) as pool:
        partial_results = pool.map(partial_sum, [CHUNK] * NUM_PROC)
    a = sum(partial_results)
    results['Pool+map'] = time.time() - start
    print(f"     결과: {a:,}, 시간: {results['Pool+map']:.2f}초\n")
    
    # 3. Array 사용
    print("[3/4] Array 사용...")
    shared_arr = mp.Array('l', NUM_PROC)  # 'l' = long 정수
    start = time.time()
    procs = [
        mp.Process(target=add_to_array, args=(shared_arr, i, CHUNK))
        for i in range(NUM_PROC)
    ]
    for p in procs:
        p.start()
    for p in procs:
        p.join()
    a = sum(shared_arr)
    results['Array'] = time.time() - start
    print(f"     결과: {a:,}, 시간: {results['Array']:.2f}초\n")
    
    # 4. Value + Lock (느림 - 테스트용으로 축소)
    print("[4/4] Value + Lock (1000만 회로 축소)...")
    TEST_COUNT = 10_000_000
    shared_val = mp.Value('l', 0)
    lock = mp.Lock()
    chunk_small = TEST_COUNT // NUM_PROC
    start = time.time()
    procs = [
        mp.Process(target=add_with_lock, args=(shared_val, lock, chunk_small))
        for _ in range(NUM_PROC)
    ]
    for p in procs:
        p.start()
    for p in procs:
        p.join()
    results['Value+Lock'] = (time.time() - start) * 10  # 1억 기준 추정
    print(f"     결과: {shared_val.value:,}, 추정 시간(1억): {results['Value+Lock']:.2f}초\n")
    
    # 결과 요약
    print("=" * 50)
    print("📊 성능 비교 요약")
    print("=" * 50)
    base = results['순차 실행']
    for method, elapsed in results.items():
        speedup = base / elapsed
        bar = "█" * int(speedup * 5)
        print(f"{method:15} : {elapsed:6.2f}초 | 속도: {speedup:.2f}x {bar}")

if __name__ == "__main__":
    benchmark_all()
```
